Que se hizo

Partes principales del proyecto

01

Servidor Express

El servidor recibe solicitudes HTTP, usa middleware global y conecta rutas separadas para usuarios, tickets, Swagger y autenticacion.

02

Rutas REST

Las rutas organizan operaciones CRUD para listar, crear, actualizar y eliminar datos usando URLs claras.

03

Controladores

Los controladores contienen la logica que consulta MongoDB y devuelve respuestas JSON con codigos HTTP adecuados.

04

Modelos Mongoose

Los modelos definen la forma de los documentos de usuarios y tickets dentro de MongoDB.

05

Validaciones

Antes de guardar informacion, se validan campos requeridos como origen, destino, fechas, email y datos personales.

06

OAuth y Swagger

Passport Google OAuth protege las rutas reales y Swagger documenta la API para pruebas tecnicas.
